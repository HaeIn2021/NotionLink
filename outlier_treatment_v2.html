이상치 판단 및 처리

🔑  핵심 원칙:  "물리가 먼저, 통계가 나중"
통계적으로 이상해 보여도 강수–유출 메커니즘으로 설명되면 정상 데이터입니다.
반대로, 통계 범위 내에 있어도 물리적으로 설명 불가하면 이상치입니다.[1]
1 분석 전 필수 확인 — 수문학적 맥락
⚠ 통계 분석 이전에 아래 수문학적 맥락을 먼저 파악해야 합니다. 통계만으로는 홍수 첨두를 이상치로, 센서 고착값을 정상으로 오판할 수 있습니다.[2]

2 수문학적 이상치 유형 분류 [5]
수문 데이터 이상치는 원인과 처리 방법이 다른 4가지 유형으로 구분합니다.
유형 식별 없이 일괄 제거·대체하면 중요한 수문 정보를 잃을 수 있습니다.
Spike단일 시점 급등·급락
전후 값과 단절된 단발성 이상값. 수위 변화가 물리적으로 불가능한 수준.
주요 원인
센서 부유물 충격, 통신 오류, 비트 오류
판단 기준
|dH/dt| > 유역별 물리 최대값 (예: 소하천 0.5 m/h)
즉시 플래깅 QC = 2
Shift기준선 급변
특정 시점부터 수위 레벨이 갑자기 이동. 이후 패턴은 유사하지만 기준이 다름.
주요 원인
센서 위치 변동, 하상 변동, 영점(Zero) 이탈
판단 기준
강수 이벤트 없는 시점에 갑작스러운 수위 레벨 이동
BM 이력 확인 QC = 1~2
Drift점진적 이탈
서서히 비현실적인 방향으로 증가 또는 감소. 장기간 누적되므로 발견이 늦음.
주요 원인
센서 fouling(오염), 침적물 누적, 전압 저하
판단 기준
STL 분해 잔차 증가 추세, 인접 관측소와 장기 乖離
STL 분석 QC = 1~2
Flatline동일값 반복 (고착)
일정 값이 수 시간~수일 동안 지속. 갈수기 저수위 구간에서 탐지 어려움.
주요 원인
센서 동결, 통신 단절, 동일 패킷 반복 전송
판단 기준
동일값 ≥ 3시간 연속 → 의심 / ≥ 6시간 → 오류 강력 의심[6]
센서 점검 필요 QC = 2
3 이상치 판단 방법론 — 5단계 우선순위
1
물리 기반 임계값 최우선
수위 변화율 |dH/dt| > 물리적 최대 가능 상승률 → 즉시 이상치
최대 가능 상승률 = 유역면적 × 극한강수량(PMF)으로 추정[1]
예: 유역 100 km² 소하천 → 시간당 0.5 m 이상 상승은 물리적으로 불가능
2
강수–수위 상관 검증 (Rainfall–Runoff Consistency)[2]
강수량 = 0 && 수위 급등 → 이상치 후보 ①
강수량 > 임계값 && 수위 무반응 → 이상치 후보 ②
유역 집중시간(Tc) 이내 수위 변화 기대량 범위 이탈 → 플래깅
조건기대 결과판단
강수 없음 + 수위 급등변화 없음이 정상이상치 후보 ①
강수 있음 + 수위 무반응Tc 이내 상승 기대이상치 후보 ②
강수 후 Tc 이내 수위 상승수문곡선 패턴 일치정상 (홍수 피크)
3
통계적 탐지 — 계절별 분리 필수 ⚠ 주의사항 있음
수위 데이터는 정규분포를 따르지 않습니다(GEV·Gumbel 극값 분포).[7] 홍수기와 갈수기를 같은 모집단으로 보면 정상 홍수 피크도 이상치로 오판합니다. Z-score 임계값은 일반 3σ가 아닌 3.5~4σ를 권장합니다.
4
시계열 분해(STL) 잔차 분석[8]
수위(t) = 추세 + 계절성 + 잔차 → 잔차가 큰 시점이 이상치 후보
단, 홍수 피크를 이상치로 오판하지 않도록 강수 데이터 교차 검증 필수
5
공간적 교차 검증 (Spatial Cross-validation) 가장 강력[9]
동일 수계 내 상·하류 관측소 수위 패턴과 불일치 → 이상치 가능성 높음
상류 수위 상승 없이 하류만 급등 → 센서 오류로 강하게 의심
광역 강우 이벤트 시 인접 관측소와 동반 상승 → 정상으로 판단
💡 인접 관측소 데이터가 있다면 공간 교차 검증을 최우선으로 활용하세요. 단일 통계 방법 대비 오탐지율이 현저히 낮습니다.
4 일반적 이상치 처리 방법
플래그 표시 실무 기본
원본 유지 + 의심 구간 기록
적용 상황
이상치 여부 불확실하거나 원본 보존 필요 시
방법
QC 컬럼 추가 (0=정상, 1=의심, 2=오류)
원본 보존 수문 표준[4]
삭제 (제거)
이상치 값을 NaN으로 대체 후 결측 처리
적용 상황
센서 오류·입력 오류가 명확하고 QC=2인 경우
주의
삭제 후 결측치 처리 방침과 연계. 홍수 피크 삭제 금지.
QC=2 한정 값 → NaN
대체 (대입)
이상값을 통계값 또는 유도값으로 교체
적용 상황
근거 있는 대체값 확보 시 (Rating Curve, 인접 관측소 등)
주의
단순 평균·중앙값 대체는 수문 데이터에 부적합[9]
근거 필수 플래그 병행

개요
시간 기반 보간이란?
일반 선형 보간은 행 번호 기준으로 값을 채웁니다.
시간 기반 보간은 실제 시간 간격을 기준으로 값을 채웁니다.
날짜 간격이 일정하지 않은 수위 데이터에 적합합니다.
날짜수위 (m)비고

01-02?결측
01-04?결측 (2일 공백)
01-050.45정상
1단계
시간 간격 확인
01-01 → 01-05 사이 총 시간 간격은 4일입니다.
결측 위치는 01-02(1일 후), 01-04(3일 후)입니다.
전체 간격 = 01-05 − 01-01 = 4일 01-02 비율 = 1일 / 4일 = 0.25 01-04 비율 = 3일 / 4일 = 0.75
2단계
추정값 계산
시작값 0.37m, 종료값 0.45m, 차이 0.08m
각 비율에 차이를 곱해 더하면 추정값이 나옵니다.
01-02 추정 = 0.37 + 0.25 × 0.08 = 0.39m 01-04 추정 = 0.37 + 0.75 × 0.08 = 0.43m
3단계
결과 확인 & 플래그 표시
결측값이 채워졌습니다. 추정값은 파란색으로 표시됩니다.
실무에서는 flag 컬럼으로 원본과 추정값을 반드시 구분합니다.

💡 핵심 원칙 요약
어떤 방법을 쓰든 추정값임을 반드시 QC 플래그로 표시해야 합니다. 원본값과 추정값을 구분해서 관리하는 것이 수문 실무의 기본입니다. 이상치는 제거가 아닌 플래그 → 검토 → 최종 판단의 3단계로 처리합니다.[1]
